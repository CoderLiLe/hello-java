# 设计模式

## 1. 设计模式概述

### 1.1 六大原则
- **单一职责**：一个类只做一件事
- **开闭原则**：对扩展开放，对修改关闭
- **里氏替换**：子类可替换父类
- **接口隔离**：接口职责单一
- **依赖倒置**：依赖抽象而非具体
- **迪米特法则**：最少知道原则

## 2. 创建型模式

### 2.1 单例模式 Singleton
保证一个类仅有一个实例。

```java
// 饿汉式
public class Singleton {
    private static final Singleton INSTANCE = new Singleton();
    private Singleton() {}
    public static Singleton getInstance() { return INSTANCE; }
}

// 双重检查锁
public class Singleton {
    private static volatile Singleton instance;
    private Singleton() {}
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

// 静态内部类
public class Singleton {
    private Singleton() {}
    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }
    public static Singleton getInstance() { return Holder.INSTANCE; }
}

// 枚举
public enum Singleton {
    INSTANCE;
    public void doSomething() {}
}
```

### 2.2 工厂模式 Factory
```java
// 简单工厂
interface Product {}
class ProductA implements Product {}
class ProductB implements Product {}
class SimpleFactory {
    public static Product create(String type) {
        switch (type) {
            case "A": return new ProductA();
            case "B": return new ProductB();
            default: throw new IllegalArgumentException();
        }
    }
}

// 工厂方法
interface Factory {
    Product create();
}
class FactoryA implements Factory {
    public Product create() { return new ProductA(); }
}
class FactoryB implements Factory {
    public Product create() { return new ProductB(); }
}

// 抽象工厂
interface AbstractFactory {
    ProductA createA();
    ProductB createB();
}
```

### 2.3 建造者模式 Builder
```java
public class Computer {
    private String cpu;
    private String ram;
    private String disk;
    
    public static class Builder {
        private Computer computer = new Computer();
        public Builder cpu(String cpu) { computer.cpu = cpu; return this; }
        public Builder ram(String ram) { computer.ram = ram; return this; }
        public Builder disk(String disk) { computer.disk = disk; return this; }
        public Computer build() { return computer; }
    }
}
// 使用
Computer c = new Computer.Builder().cpu("i7").ram("16G").disk("512G").build();
```

### 2.4 原型模式 Prototype
```java
public class Prototype implements Cloneable {
    public Prototype clone() {
        return (Prototype) super.clone();
    }
}
```

## 3. 结构型模式

### 3.1 适配器模式 Adapter
```java
// 类适配器
interface Target { void request(); }
class Adaptee { public void specificRequest() {} }
class Adapter extends Adaptee implements Target {
    public void request() { specificRequest(); }
}

// 对象适配器
class ObjectAdapter implements Target {
    private Adaptee adaptee;
    public void request() { adaptee.specificRequest(); }
}
```

### 3.2 代理模式 Proxy
```java
interface Subject { void request(); }
class RealSubject implements Subject {
    public void request() { /* 实际逻辑 */ }
}
class Proxy implements Subject {
    private RealSubject real;
    public void request() {
        before();
        real.request();
        after();
    }
}
```

### 3.3 装饰器模式 Decorator
```java
interface Component { void operation(); }
class ConcreteComponent implements Component {
    public void operation() {}
}
abstract class Decorator implements Component {
    protected Component component;
    public void operation() { component.operation(); }
}
class ConcreteDecorator extends Decorator {
    public void operation() {
        super.operation();
        addedBehavior();
    }
}
```

### 3.4 其他结构型模式
- **桥接模式 Bridge**：抽象与实现分离
- **组合模式 Composite**：树形结构
- **外观模式 Facade**：统一接口
- **享元模式 Flyweight**：共享对象

## 4. 行为型模式

### 4.1 策略模式 Strategy
```java
interface Strategy {
    int execute(int a, int b);
}
class AddStrategy implements Strategy {
    public int execute(int a, int b) { return a + b; }
}
class SubtractStrategy implements Strategy {
    public int execute(int a, int b) { return a - b; }
}
class Context {
    private Strategy strategy;
    public int executeStrategy(int a, int b) { return strategy.execute(a, b); }
}

// Lambda 写法
Context ctx = new Context((a, b) -> a + b);
```

### 4.2 观察者模式 Observer
```java
interface Observer { void update(String msg); }
class ConcreteObserver implements Observer {
    public void update(String msg) { System.out.println(msg); }
}
class Subject {
    private List<Observer> observers = new ArrayList<>();
    public void attach(Observer o) { observers.add(o); }
    public void notifyAll(String msg) {
        observers.forEach(o -> o.update(msg));
    }
}
```

### 4.3 模板方法 Template
```java
abstract class AbstractClass {
    public final void templateMethod() {
        step1();
        step2();
        hook();
    }
    protected abstract void step1();
    protected abstract void step2();
    protected void hook() {} // 可选步骤
}
```

### 4.4 其他行为型模式
- **责任链 Chain**：请求逐级处理
- **命令 Command**：请求封装为对象
- **迭代器 Iterator**：遍历集合
- **状态 State**：状态驱动行为
- **备忘录 Memento**：状态快照
- **中介者 Mediator**：解耦多对象交互
- **访问者 Visitor**：元素操作分离
- **解释器 Interpreter**：语法解释
