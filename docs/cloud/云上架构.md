# 云服务篇-云上架构与场景方案实战

## 1. 云上架构概览

### 1.1 核心设计原则
- **高可用**：多可用区部署
- **弹性伸缩**：按需扩展
- **安全合规**：多层次安全防护
- **成本优化**：合理选择资源
- **可运维**：监控、日志、自动化

### 1.2 典型云上架构
```
               CDN
                |
         SLB（负载均衡）
        /         \
    Web1          Web2        <-- 弹性伸缩组
        \         /
          App1 - App2         <-- RPC/消息队列
            |       |
    Redis  MySQL  MongoDB     <-- 高可用/读写分离
```

## 2. 核心服务

### 2.1 计算
| 服务 | 说明 | 适用场景 |
|------|------|---------|
| ECS | 云服务器 | 通用场景 |
| 容器服务 | K8s 集群 | 微服务 |
| Serverless | 函数计算 | 事件驱动 |
| 弹性伸缩 | 自动扩缩容 | 波峰波谷 |

### 2.2 存储
- **对象存储（OSS）**：静态文件、备份归档
- **文件存储（NAS）**：共享文件系统
- **块存储**：云盘

### 2.3 网络
- **VPC**：私有网络隔离
- **SLB**：负载均衡
- **NAT 网关**：公网访问
- **VPN/专线**：混合云

### 2.4 数据库
- **RDS**：MySQL / PostgreSQL / SQL Server
- **Redis**：缓存
- **MongoDB**：文档数据库
- **DRDS / PolarDB**：分布式数据库

## 3. 场景方案

### 3.1 高并发场景
```
CDN 加速 -> WAF 防护 -> SLB 分发 -> 
Redis 缓存 -> 异步 MQ -> 数据库读写分离
```

### 3.2 容灾架构
| 方案 | RTO | RPO | 成本 |
|------|-----|-----|------|
| 同城双活 | 分钟级 | 秒级 | 中 |
| 两地三中心 | 分钟级 | 分钟级 | 高 |
| 异地多活 | 秒级 | 零 | 最高 |

### 3.3 混合云
- 弹性伸到公有云
- 灾备到公有云
- 研发测试在公有云

## 4. 自动化运维

### 4.1 IaC（基础设施即代码）
```yaml
# Terraform 示例
resource "alicloud_instance" "web" {
  instance_type   = "ecs.g6.large"
  image_id        = "centos_7_9_x64"
  security_groups = [alicloud_security_group.web.id]
  vswitch_id      = alicloud_vswitch.public.id
  user_data       = <<-EOF
    #!/bin/bash
    yum install -y nginx
    systemctl start nginx
  EOF
}
```

### 4.2 CI/CD 流程
```
代码提交 -> 自动构建 -> 自动测试 -> 
自动部署（灰度）-> 自动验证 -> 全量发布
```

### 4.3 容器化迁移

COPY app.jar /app.jar

CMD ["/startup.sh"]

## 5. 安全体系

### 5.1 安全层
- **网络层**：安全组、ACL、WAF
- **应用层**：身份认证、权限管理
- **数据层**：加密、脱敏、备份
- **主机层**：漏洞扫描、基线检查

### 5.2 合规
- 等保 2.0
- GDPR
- SOC
- PCI DSS
