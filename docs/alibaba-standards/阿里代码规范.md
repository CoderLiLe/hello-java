# 阿里系常用代码规范及工具

## 1. 编程规约

### 1.1 命名风格
- **类名**：UpperCamelCase（DO / VO / DTO / BO / POJO 等除外）
- **方法名**：lowerCamelCase
- **常量**：UPPER_SNAKE_CASE
- **抽象类**：Abstract 或 Base 开头
- **异常类**：Exception 结尾
- **测试类**：Test 结尾

### 1.2 常量定义
- 魔法值（魔数）不允许出现在代码中
- long 或 Long 赋值时使用大写 L（如 `100L`）
- 不同作用域常量放在不同类中

### 1.3 代码格式
- 缩进：4 空格（禁止 Tab）
- 大括号：K&R 风格（左括号前不换行）
- 单行字符限制：120 或 150

### 1.4 OOP 规约
- **避免通过对象引用访问静态变量/方法**（类名调用）
- **覆写方法加 @Override**
- **集合泛型指定类型**
- **equals 常量放前面**：`"str".equals(obj)`
- **所有 POJO 类实现 toString**
- **尽量不使用 BeanUtils 深拷贝**

### 1.5 集合处理
```java
// 集合初始化指定大小
Map<String, String> map = new HashMap<>(16);
List<String> list = new ArrayList<>(100);

// Arrays.asList 返回的不可变
List<String> list = new ArrayList<>(Arrays.asList("a", "b"));

// subList 不可序列化，强引用视图
// 集合转数组使用 toArray(T[] array)
String[] array = list.toArray(new String[0]);
```

### 1.6 并发处理
- **线程池不允许使用 Executors 创建**（使用 ThreadPoolExecutor）
- SimpleDateFormat 是线程不安全的
- 高并发时同步调用考虑锁粒度

### 1.7 日期处理
- JDK8 使用 `java.time` 包（`LocalDate`, `LocalTime`, `LocalDateTime`）

### 1.8 异常处理
- **不要捕获 RuntimeException**
- **异常不要用来做流程控制**
- **finally 块中不要 return**
- **事务场景中捕获异常后需要回滚**

## 2. 数据库规约

### 2.1 建表规约
- 表名、字段名：小写下划线
- 禁用保留字
- **每张表必须有 id、create_time、update_time**
- id 类型：bigint unsigned
- 小数类型：decimal（禁止 float/double）
- 字段尽量 NOT NULL

### 2.2 索引规约
- 索引不超过 5 个
- 复合索引区分度最大的在最前面
- JOIN 字段类型需一致
- 不要在 SQL 中对列做运算

### 2.3 SQL 规约
- 不要使用 `SELECT *`
- `COUNT(*)` 统计行数
- 分页查询 count 使用 `COUNT(1)` 而非 `COUNT(列)`

## 3. 工程规约

### 3.1 应用分层
```
Controller（表现层） -> Service（业务层） -> Manager（管理） -> DAO（数据层）
```

### 3.2 各层命名
- **Controller**：XxxController
- **Service 接口**：IXxxService
- **Service 实现**：XxxServiceImpl
- **Mapper**：XxxMapper
- **数据对象**：XxxDO
- **传输对象**：XxxDTO
- **视图对象**：XxxVO

### 3.3 包命名
```
com.公司名.项目名.模块名.分层名
示例：com.alibaba.app.user.service
```

- Alibaba Java Coding Guidelines

## 5. 安全规约

- 用户敏感数据加密存储
- SQL 参数使用预编译/参数化查询
- 前端提交数据需要后端校验
- 防止 XSS、CSRF 攻击
- 日志中不记录敏感信息
- API 接口需要鉴权和限流
