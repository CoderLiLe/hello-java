# 容易忽略的测试

## 1. 测试种类

### 1.1 单元测试
```java
@SpringBootTest
class UserServiceTest {
    @Autowired
    private UserService userService;
    
    @Test
    void testCreateUser() {
        UserDTO dto = new UserDTO();
        dto.setName("test");
        User user = userService.create(dto);
        assertNotNull(user.getId());
        assertEquals("test", user.getName());
    }
    
    @Test
    void testDuplicateName() {
        assertThrows(BusinessException.class, () -> {
            userService.create(new UserDTO("existing"));
        });
    }
}
```

### 1.2 集成测试
```java
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
class UserControllerTest {
    @Autowired
    private TestRestTemplate restTemplate;
    
    @Test
    void testCreateUser() {
        ResponseEntity<UserVO> response = restTemplate.postForEntity(
            "/api/users", new UserDTO("test"), UserVO.class);
        assertThat(response.getStatusCode()).isEqualTo(HttpStatus.CREATED);
    }
}
```

### 1.3 持久层测试
```java
@DataJpaTest
class UserRepositoryTest {
    @Autowired
    private UserRepository userRepository;
    
    @Test
    void testFindByName() {
        userRepository.save(new User("test"));
        User user = userRepository.findByName("test");
        assertThat(user).isNotNull();
    }
}
```

## 2. 容易被忽略的测试点

### 2.1 边界值测试
- 数值边界：0、负数、最大值、最小值
- 字符串边界：空串、null、超长字符串
- 集合边界：空集合、单元素、大量元素
- 时间边界：时区、闰年、夏令时、跨年

### 2.2 并发测试
```java
@Test
void testConcurrentUpdate() throws Exception {
    int threadCount = 10;
    CountDownLatch latch = new CountDownLatch(threadCount);
    AtomicInteger success = new AtomicInteger(0);
    
    for (int i = 0; i < threadCount; i++) {
        new Thread(() -> {
            try {
                stockService.decrease(1L, 1);
                success.incrementAndGet();
            } catch (Exception e) {
                // 并发冲突
            } finally {
                latch.countDown();
            }
        }).start();
    }
    latch.await();
    assertThat(success.get()).isLessThanOrEqualTo(threadCount);
}
```

### 2.3 幂等性测试
```java
@Test
void testIdempotent() {
    // 多次提交相同请求
    for (int i = 0; i < 3; i++) {
        Result result = orderService.create(orderDTO);
        if (i == 0) {
            assertTrue(result.isSuccess());
        } else {
            // 后续请求应幂等
            assertTrue(result.isDuplicate());
        }
    }
}
```

### 2.4 事务测试
```java
@Test
void testTransactionRollback() {
    assertThrows(Exception.class, () -> {
        orderService.createWithInvalidData(orderDTO);
    });
    // 验证数据回滚
    assertNull(orderMapper.selectByOrderNo(orderDTO.getOrderNo()));
}
```

### 2.5 安全性测试
- SQL 注入测试
- XSS 测试
- CSRF 测试
- 越权访问测试（水平/垂直）
- 敏感信息泄露

### 2.6 性能测试
```java
@Test
void testPerformance() {
    long start = System.currentTimeMillis();
    for (int i = 0; i < 100; i++) {
        userService.complexQuery(userId);
    }
    long elapsed = System.currentTimeMillis() - start;
    assertThat(elapsed).isLessThan(1000); // P99 < 10ms
}
```

### 2.7 兼容性测试
- 浏览器兼容
- 数据库版本兼容
- JDK 版本兼容
- 移动端适配

### 2.8 数据一致性测试
- 缓存与数据库一致性
- 主从数据一致性
- ES 与 DB 数据同步

## 3. 测试金字塔

```
    /\
   /  \       E2E 测试（少量）
  /    \
 / UI   \     集成测试（适量）
/________\
| Unit   |    单元测试（大量）
|  Test  |
|________|
```

## 4. 测试策略建议

- 单元测试覆盖率 > 80%
- 核心业务逻辑全覆盖
- 异常场景覆盖完整
- 自动化测试集成到 CI
- 定期进行混沌工程测试
