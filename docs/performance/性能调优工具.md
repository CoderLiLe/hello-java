# 性能调优工具

## 1. JDK 内置工具

### 1.1 jps（JVM Process Status）
```bash
jps                          # 显示 Java 进程
jps -l                       # 显示完整包名
jps -v                       # 显示 JVM 参数
```

### 1.2 jstat（JVM Statistics Monitoring）
```bash
jstat -gc <pid> 1000 10      # 每 1 秒打印 GC 信息，共 10 次
jstat -gcutil <pid> 1000     # GC 百分比
jstat -gccause <pid>         # GC 原因
```

### 1.3 jinfo（Configuration Info）
```bash
jinfo <pid>                  # 查看 JVM 参数
jinfo -flag <name> <pid>     # 查看特定参数
jinfo -flag +<name> <pid>    # 动态启用参数
```

### 1.4 jmap（Memory Map）
```bash
jmap -heap <pid>             # 堆信息
jmap -histo <pid>            # 对象统计
jmap -dump:format=b,file=heap.hprof <pid>  # 堆转储
```

### 1.5 jstack（Stack Trace）
```bash
jstack <pid>                 # 线程堆栈
jstack -l <pid>              # 长格式（含锁信息）
jstack -m <pid>              # 混合模式（含 C++ 帧）
```

### 1.6 jcmd
```bash
jcmd <pid> help              # 可用命令
jcmd <pid> VM.version        # JVM 版本
jcmd <pid> GC.heap_dump dump.hprof  # 堆转储
jcmd <pid> Thread.print      # 线程栈
```

### 2.4 MAT（Memory Analyzer Tool）
- 堆转储分析
- 内存泄漏检测
- 大对象分析
- GC Roots 追踪

#### 使用示例
```bash
# 生产环境获取堆转储
jmap -dump:live,format=b,file=heap.hprof <pid>

# 使用 MAT 分析
# 1. 打开 heap.hprof
# 2. Leak Suspects Report -> 查找泄漏嫌疑
# 3. Dominator Tree -> 查看大对象
# 4. GC Roots -> 追踪引用链
```

## 3. 性能分析

### 3.1 CPU 分析
```bash
# 找出 CPU 最高的 Java 线程
top -H -p <pid>
printf "%x\n" <tid>           # 转十六进制
jstack <pid> | grep -A 30 <tid-hex>
```

### 3.2 内存分析
```bash
# 检查 OOM
jstat -gcutil <pid> 1000

# 生成堆转储
jmap -dump:live,format=b,file=heap.hprof <pid>

# 检查 GC 情况
jstat -gc <pid> 1000 10
```

### 3.3 线程分析
```bash
# 死锁检测
jstack -l <pid> | grep -A 30 "deadlock"

# 线程状态分布
jstack <pid> | grep "java.lang.Thread.State" | sort | uniq -c
```

### 3.4 GC 日志分析
```bash
# JVM 参数开启 GC 日志
-XX:+PrintGCDetails
-XX:+PrintGCDateStamps
-Xloggc:/path/to/gc.log

# 工具：GCeasy、GCViewer
```

## 4. Linux 性能工具

### 4.1 CPU
- `top` / `htop`：实时进程监控
- `vmstat`：系统整体性能
- `perf`：性能事件采样
- `mpstat`：多核 CPU 统计

### 4.2 内存
- `free -h`：内存使用
- `vmstat 1`：内存交换情况
- `slabtop`：内核 slab 信息

### 4.3 磁盘 I/O
- `iostat -x 1`：磁盘 I/O 详细
- `iotop`：进程 I/O 监控
- `dstat`：全能统计

### 4.4 网络
- `netstat` / `ss`：网络连接
- `tcpdump`：抓包分析
- `nload` / `iftop`：实时流量
- `sar -n DEV 1`：网络统计

## 5. 数据库调优

### 5.1 MySQL
```sql
-- 慢查询日志
SET GLOBAL slow_query_log = ON;
SET GLOBAL long_query_time = 1;

-- 查看执行计划
EXPLAIN SELECT * FROM user WHERE id = 1;
-- 查看索引使用
SHOW INDEX FROM user;
```

### 5.2 Redis

## 7. 调优步骤
1. **明确目标**：延迟/吞吐量/资源
2. **性能基准**：建立基线数据
3. **问题定位**：CPU/内存/IO/锁
4. **瓶颈分析**：使用对应工具
5. **方案实施**：代码/配置/架构优化
6. **效果验证**：对比基准数据
7. **持续监控**：建立监控体系
